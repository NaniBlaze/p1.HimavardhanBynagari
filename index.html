Microwave Report

Current design of the Microwave :

Microwave Interface

This Microwave has basic operations which is basically neccery for every user. It have special features like saving favroutie cooking styles.

Instant Cooking Operation

This Microwave has instant cooking options, which helps user to cook basic items which are commonly used in this region.

Manual Cooking Operation

This Microwave has provided manual cooking. To reduce time consumtion on this feature it is placed in highlited block, since it is most commonly used.

The Probems found in the current design are below:

Problem 1 - The highlighted block is not well encapsulated

The "Time Cook", "Auto Cook" and "Auto Deforst" are provided in same block.
Since Most of the options in the highlighted block are "Automated" and "Time Cook" is a manual option, it often confuses the user when he wanted to cook manually.

Problem 2 - Automatic buttons needed final approval

The "Auto Cook" and "Auto Defrost" are designed to reduce time of the user.
Even though these buttons are clicked, the machine is haulted for the final approval.
The automatic feature is to help user to reduce the time but this feature take equal amount of time as normal cooking method.

Problem 3 - The Timer and Time Cook are similar

The "Timer" and "Time Cook" have features similar and as a new user to the machine often confuses with these buttons.
Since the design is not specific to the clock operations and cooking operations, user needs time to get used to this machine.

Problem 4 - Vent Fan is not specific

The "Vent Fan" button starts vent fan with three levels as (off-high), (high-low) and (low-off).
After clicking the button, the current level is displayed for only a second and disappear.
If user want to know the current level, this design is not helpful. To get to the requried level user has to complete the whole cycle.

Common Uses of Microwave:

  • Popcorn
  • Beverage
  • Frozen Vegetables / Pizza
  • Dinner Plate
  • Beverage

The mentioned are the frequent operations used daily.
The Current design had alredy provided the best way possible to help the user in making it easy and faster for these operations.

Rare Uses of Microwave:

  • Frozen Meet
  • Timer
  • Power Level
  • Warm

The mentioned are the rarely used operations.
The Current design is not very compatable for these rarely used operations .
As mentioned above (In Problems), to defrost the frozen meet it will take around 45 minitues and it has to be cooked as user specifications. The current interface is not helpful.
The "Timer" is very confusing with the "Time Cook" as it has similar structure and it is not proper alligned as encapsulated.
The "Power Level" and "Warm" are very rarely used since the temparature is normally fixed so that user will know the cooking time and the current design is well established.

User Common Sequence of Actions

  1. User open the door and place the item inside microwave.
  2. Select the required option.
  3. If user selects default options or favourite options, then the machine starts automatically.
  4. If user wants to manually add the time and power level he can do that with ease.
  5. The machine will notify the user with sound and with a display message ralated to the respective feedback.

Feedback

    Cooking Feedback

    Timer Feedback

    When ever the clock task's are completed the machine indicates user with this sound.

  • Display : It provides different messages in the machine screen respective to the task completed.
    Cooking Feedback

    When ever the cooking task's are completed the machine generates this message to notify user.

    Timer Feedback

    When ever the clock task's are completed the machine generates this message to notify user.

Common Mistakes

  • As I mentioned in the existing problems, the user often confuses with the "Time Cook" and "Timer" buttons because of their uneven placement in the design.
  • Since the "Time cook" is in the hilighted design of the Automatic functionality, the user might get the wrong idea of the functionality.
  • Everytime the task is ended, the machine will not automatically come to the initial position unless the user clicks the "cancel" button.
    This might confuse the user at first.

Improvement Design Plan :

This is the basic design I created is shown below.


The First Screen of this model contails basic information like "Time" and most necessary options.
  • Time : This feature shows the time in 24 hours format (optimal way for everyone) and the time is taken by the current region of the machine.

  • Timer : This features allows the user to set time and the machine will coundown the time and notify the user when time is completed. This comes handy for the user on occasional situations rather than only for cooking.

  • Cook : This option will take user to the manual cooking page where the user can enter the time required and after approval the microwave cooks the food and notifies the User.

  • Automatic Section : This section has 3 options. Which works automatically with fixed time registered in the system. These buttons does not require any further approval from user like the olds design, so this will help user to cook faster. These options work only when the door of the microwave is closed.

    The options provided in the Automatic Section are :
    • Auto Cook : This option cooks the food automatically when the door is closed for 2 minitues of time.
    • Auto Defrost : This option will Defrost the item in the microwave for 45 minitues.
    • Warm : This option will warm the food for 1 minitue.

  • Special Features : These features are linked to the microwave which are useful to the user while cooking or doing something else.
    The Special Features provided in this setion are :
    • Vent Fan : This feature turns on the vent fan and helps user to remove the cooking smoke at ease and helps in several occasions.
    • Lights : The lights are embeded to the microwave and this feature helps user to turn on and off the lights.

The "Cook" and "Timer" option will take user to different screen where user normally enter the time required for action. This data entry page design is as below.


Most of the design looks similar for "Cook" and "Timer" options so a message is displayed to the user saying "Manual Cooking" and "Timer" respectively.
In this screen the machine provides three options :
  • Start : This will start the cooking or Timer with the timer mentioned by the user. For cooking alone, if the door of microwave is open then cooking will automatically gets paused and after cloing the door it resumes automatically.
  • Cancel : This will cancel the operation and directs the user to the main scrren of microwave.
  • Clear : This feature will clear the time-input entered and allows user to enter new time.

Finally the cooking Screen, The cooking screen will show the user the cooking-percentage completed and with a designed area covering diagram representes the percentage completed. While the microwave is cooking the food or processing the timer, two options are displayed which are :
  • Pause/Resume : This buttons allows user to pause or resume the cooking and timer actions.
  • Stop : This feature allows user to stop the process either cooking or timer instantly and directs to the main screen of the microwave.

After the cooking is completed these options ("Pause/Resume" and "Stop") are removed and a new button comesup saying "Home Screen". This will direct the user to the main screen of the microwave.


Note :
Once the manual cooking option is selected the automatic cooking options from "Automatic Section" are disabled to the user.
