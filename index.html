Current design of the Microwave

Microwave Interface

This Microwave has basic operations which is basically neccery for every user. It have special features like saving favroutie cooking styles.

Instant Cooking Operation

This Microwave has instant cooking options, which helps user to cook basic items which are commonly used in this region.

Manual Cooking Operation

This Microwave has provided manual cooking. To reduce time consumtion on this feature it is placed in highlited block, since it is most commonly used.

The Probems found in the current design are below:

Problem 1

The highlited block is not well encapsulated.

The "Time Cook", "Auto Cook" and "Auto Deforst" are provided in same block.
Since Most of the options in the highlited block are "Automated" and "Time Cook" is a manual option, it offen confuses the user when he wanted to cook manually.

Problem 2

Automatic buttons needed final approval

The "Auto Cook" and "Auto Deforst" are designed to reduce time of the user.
Even though these buttons are clicked, the machine is halted for the final approval.
The automatic feature is to help user to reduce the time but this feature take equal amount of time as normal cooking method.

Problem 3

The Timer and Time Cook are similar

The "Timer" and "Time Cook" have features similar and as a new user to the machine often confuses with these buttons.
Since the design is not specific to the clock operations and cooking operations, user needs time to ge used to this machine.

Problem 4

Vent Fan is not specific

The "Set Vent Fan" button starts vent fan with three levels as (off-high), (high-low) and (low-off).
After clicking the button, the current level is displayed for only a second and disapieres.
If user want to know the current level, this design is not helpful. To get to the requried level user has to complete the whole cycle.

Rare Uses of Microwave:

  • Frozen Meet
  • Timer
  • Power Level
  • Warm

The mentioned are the Rarely used operations.
The Current design is not very compatable for these rarly used operations .
As mentioned above (In Problems), to defrost the frozen meet it will take around 45 minitues and it has to be cooked as user specifications. The current interface is not helpful.
The "Timer" is very confusing with the "Time Cook" as it has similar structure and it is not proper alligned as encapsulated.
The "Power Level" and "Warm" are very rarly used since the temparature is normally fixed so that user will know the cooking time and the current design is well established.
